# Module 3: AI Orchestration with Kestra

In this module, we learn how to orchestrate AI workflows using
[Kestra](https://kestra.io/), an open-source orchestration platform.
We start from the context problem that makes generic AI assistants
unreliable, and end with autonomous multi-agent systems that can
research, reason, and act without a fixed predetermined sequence of steps.

Flows: [flows/](flows/)


## Lessons

The lessons cover AI-powered orchestration end to end, from context
engineering to multi-agent systems.

1. [Introduction](lessons/01-intro.md) - Learning objectives, prerequisites, and why AI for workflows
2. [Context Engineering](lessons/02-context-engineering.md) - Why generic AI assistants produce incorrect Kestra flows
3. [AI Copilot](lessons/03-ai-copilot.md) - Setup and hands-on use of Kestra's built-in AI Copilot
4. [Retrieval Augmented Generation](lessons/04-rag.md) - Grounding AI responses in real data with RAG
5. [AI Agents](lessons/05-agents.md) - Autonomous task execution, available tools, and observability
6. [Multi-Agent Systems](lessons/06-multi-agent.md) - Specialized agents collaborating on complex tasks
7. [Getting Started](lessons/07-setup.md) - Kestra setup, API keys, and importing the example flows
8. [Best Practices](lessons/08-best-practices.md) - Cost, security, observability, and production readiness
